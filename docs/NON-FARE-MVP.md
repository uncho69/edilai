# Cosa NON fare nell’MVP — Lista esplicita per il dev

Nell’MVP **non** vanno sviluppati i seguenti elementi. Se qualcuno li propone (stakeholder o dev), considerarli **scope creep** e rimandare a una fase successiva.

| Non implementare | Motivo (per il team) |
|------------------|----------------------|
| **Pagamenti** | Nessun flusso di pagamento nell’MVP. |
| **Rating / recensioni** | Valutazione imprese fuori scope. |
| **Contratti** | Nessuna gestione contratti. |
| **Firma digitale** | Non richiesta nell’MVP. |
| **Comparazione automatica preventivi** | Solo invio richiesta e risposta impresa (accetta/rifiuta/sopralluogo). |
| **Gestione avanzata di più imprese per lo stesso lavoro** | Priorità alla strutturazione della richiesta, non al matching multiplo. |
| **AI che “sceglie” l’impresa** | Assegnazione richieste gestita dall’admin. |
| **Misurazioni precise o rendering** | Solo metrature approssimative e descrizioni testuali. |

**Regola:** se non è nella SPEC-MVP.md come “da fare”, non va nell’MVP.
